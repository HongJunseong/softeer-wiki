# Week4 회고

4주차에는 Docker 환경에서 Apache Spark Standalone Cluster를 구축하고, Master와 Worker가 작업을 분산 처리하는 구조를 직접 확인했습니다. 또한 NYC TLC Trip Record 데이터를 Spark DataFrame으로 처리하며 시간대별·지역별 운행량, 운행 거리, 소요 시간, 요금 등의 데이터를 집계하고 분석했습니다. 분석 과정에서는 단순한 통계 결과를 나열하는 데 그치지 않고, 전기차 충전소 데이터와 결합해 충전 인프라가 부족한 지역과 우선 투자 지역을 찾는 데이터 프로덕트로 발전시키는 방향을 고민했습니다. 이와 함께 대규모 웹 크롤링 사례를 살펴보며 수십억 개의 웹 페이지를 처리하기 위한 분산 시스템의 확장성, 병렬 처리, 중복 제거, 장애 대응의 중요성도 이해할 수 있었습니다. AWS IAM 실습을 통해 사용자, 그룹, 역할, 정책의 차이와 최소 권한 원칙을 학습하며 클라우드 환경에서의 권한 관리 방식도 익혔습니다.

다만 대용량 NYC TLC 데이터를 로컬 Spark 환경에서 처리하는 과정에서 실행 시간이 길어지고, 스키마 불일치나 라이브러리 설정 문제로 오류를 해결하는 데 많은 시간이 소요되었습니다. 또한 처음에는 데이터에서 확인되는 현상을 단순히 시각화하고 설명하는 데 집중해, 실제 의사결정에 활용할 수 있는 인사이트와 비즈니스 가치를 충분히 연결하지 못했습니다. 특히 데이터에 존재하지 않는 값을 임의로 가정하지 않으면서도 의미 있는 분석 기준을 만들고, 여러 분석 결과를 하나의 논리적인 스토리로 정리하는 것이 어려웠습니다. 다음 주차에는 Spark의 파티션, 캐시, 컬럼 선택, 스키마 지정 등 성능 최적화 방법을 복습하고, 분석을 시작하기 전에 해결하려는 문제와 의사결정 기준을 먼저 명확히 정의하겠습니다. 또한 분석 결과를 기술적인 설명에만 머무르지 않고 실제 데이터 프로덕트와 비즈니스 활용 방안으로 연결하는 연습을 이어가겠습니다.
