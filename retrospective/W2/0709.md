# 2026-07-09 회고

## Keep
- 감정분석 모델을 학습하고, 웹스크래핑한 현대자동차 유튜브 댓글에 적용하여 차종별 소비자 반응을 비교하는 전체 데이터 파이프라인을 구현했다.
- 여러 모델의 성능을 비교하고 Validation Accuracy, Validation Loss 등을 함께 고려하여 최종 모델을 선정하는 과정을 경험했다.
- 단순히 감정분석 결과를 제시하는 것에서 끝나지 않고, 차종별 반응을 비즈니스적으로 어떻게 활용할 수 있을지 고민해볼 수 있었다.

## Problem
- 학습 데이터(Twitter)와 분석 데이터(YouTube 자동차 댓글)의 도메인이 달라 실제 감정을 정확하게 반영하지 못할 가능성이 있다는 피드백을 받았다.
- 유튜브 댓글만으로는 실제 구매자의 의견인지, 단순 관심을 가진 사용자의 의견인지 구분하기 어려워 데이터의 신뢰성에 한계가 있다는 점을 느꼈다.
- WordCloud는 자주 등장하는 단어를 보여줄 뿐, 어떤 키워드가 실제로 긍정 또는 부정의 핵심 원인인지 파악하기에는 한계가 있었다.
- 차종별 긍정·부정 비율만으로는 "왜 이러한 결과가 나왔는가"에 대한 구체적인 근거를 제시하기 어려웠다.

## Try
- 실제 구매 경험을 나타내는 표현(예: 출고, 구매, 운행, 타고 있다 등)을 기준으로 댓글을 분류하여 구매자 중심의 데이터를 별도로 분석해보겠다.
- YouTube뿐만 아니라 자동차 커뮤니티나 리뷰 사이트의 데이터를 함께 수집하여 데이터의 다양성과 신뢰성을 높여보겠다.
- WordCloud 대신 키워드 카테고리화, Topic Modeling(BERTopic, LDA 등)과 같은 기법을 활용하여 소비자가 어떤 요소(디자인, 가격, 성능, 실내공간 등)에 대해 긍정 또는 부정적인 반응을 보였는지 더 구체적으로 분석해보겠다.
- 단순 감정 비율 비교를 넘어 판매량, 가격, 경쟁 차종 등의 데이터를 함께 활용하여 실제 비즈니스 의사결정에 도움이 되는 인사이트를 도출하는 방향으로 프로젝트를 발전시켜보겠다.
