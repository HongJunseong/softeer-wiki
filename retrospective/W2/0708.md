# 2026-07-08 회고

## Keep
- Dockerfile과 Docker Compose를 활용하여 애플리케이션 실행 환경을 컨테이너로 구성하는 과정을 익혔다.
- 로컬에서 Docker 이미지를 빌드한 후 AWS ECR에 Push하고, EC2에서 Pull하여 실행하는 배포 과정을 직접 수행하였다.
- Docker 이미지와 ECR, EC2가 어떤 역할을 하는지 이해하며 컨테이너 기반 배포의 전체 흐름을 정리할 수 있었다.

## Problem
- ECR 로그인 과정과 이미지 태그 설정, Push/Pull 과정에서 명령어의 의미를 이해하는 데 시간이 걸렸다.
- AWS Region이 ECR과 EC2에 어떤 영향을 주는지 헷갈렸고, Region이 다를 경우 발생하는 문제를 이해하는 데 어려움이 있었다.
- Docker Compose와 Docker 명령어의 역할 차이, `.env` 파일이 언제 필요한지 명확하게 구분하는 데 어려움이 있었다.

## Try
- Docker 이미지 생성부터 ECR 업로드, EC2 배포까지의 과정을 처음부터 다시 실습하며 배포 흐름을 복습하겠다.
- Docker, ECR, EC2가 각각 어떤 역할을 담당하는지 개념을 명확히 이해하겠다.
- 배포 과정에서 사용하는 주요 명령어와 목적을 함께 정리하여 다음 프로젝트에서는 스스로 배포를 진행할 수 있도록 연습하겠다.
