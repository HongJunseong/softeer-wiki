# 2026-07-14 회고

## Keep
- Docker Compose를 활용해 Multi-Node Hadoop Cluster(NameNode, DataNode, YARN)를 직접 구축하고 정상적으로 실행했다.
- `core-site.xml`, `hdfs-site.xml`, `mapred-site.xml`, `yarn-site.xml` 등 주요 설정 파일을 수정하며 각 설정이 클러스터 동작에 어떤 영향을 주는지 이해할 수 있었다.

## Problem
- 컨테이너의 hostname과 Hadoop 설정 값이 일치하지 않아 NameNode와 ResourceManager가 정상적으로 실행되지 않는 문제를 겪었다.
- 각 XML 설정 항목이 어떤 역할을 하는지 처음에는 명확히 이해하지 못해 문제 원인을 찾는 데 시간이 걸렸다.

## Try
- Hadoop 주요 설정 파일의 핵심 속성과 역할을 따로 정리하여 설정 오류가 발생했을 때 빠르게 원인을 찾을 수 있도록 하겠다.
- Multi-Node 환경을 다시 구축하며 설정을 직접 변경해 보고, 설정 변경에 따른 동작 차이를 확인해 클러스터 구조를 더 깊이 이해하겠다.
