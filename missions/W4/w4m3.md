# Crawling a Billion Web Pages in Just Over 24 Hours

## 1. 대규모 웹 크롤링 시스템 분석

이번 미션에서는 AI2(Allen Institute for AI)의 대규모 웹 크롤링 사례를 분석하였다.

- 약 24시간 동안 10억 개 이상의 웹 페이지를 수집한 크롤링 시스템을 살펴보았다.
- 대규모 분산 환경에서 웹 크롤링을 수행하기 위한 전체 구조를 이해하였다.

대규모 웹 데이터를 빠르게 수집하기 위해서는 단순한 크롤러가 아니라 분산 시스템이 필요하다는 점을 이해할 수 있었다.

## 2. 분산 크롤링 구조 이해

논문에서 제시한 크롤링 구조와 성능 최적화 방법을 학습하였다.

- 여러 Worker가 동시에 웹 페이지를 수집하는 분산 크롤링 방식을 확인했다.
- URL 관리, 중복 제거, 장애 대응 등 대규모 크롤링에서 필요한 요소들을 살펴보았다.
- 네트워크와 저장소를 효율적으로 활용하여 처리 성능을 높이는 방법을 이해하였다.

분산 환경에서는 크롤링 속도뿐만 아니라 안정성과 확장성 또한 중요한 요소임을 배울 수 있었다.

## 3. 대규모 데이터 처리의 중요성

이번 사례를 통해 대규모 데이터 수집 시스템이 어떻게 설계되는지 확인하였다.

- 웹 크롤링 시스템의 전체 동작 과정을 이해했다.
- 분산 시스템을 활용해 방대한 데이터를 효율적으로 수집하는 방법을 학습했다.
- 실제 서비스에서 사용하는 대규모 데이터 수집 아키텍처를 살펴볼 수 있었다.

이번 미션을 통해 대규모 웹 크롤링 시스템의 구조와 분산 처리 기술의 중요성을 이해할 수 있었다.
