# Apache Spark Standalone Cluster on Docker

## 1. Spark Standalone Cluster 구축

이번 미션에서는 Docker를 활용하여 Apache Spark Standalone Cluster를 구축하였다.

- Spark Master와 Worker를 각각 컨테이너로 구성했다.
- Docker Compose를 이용해 클러스터를 실행하고 노드 간 연결을 확인했다.

Spark의 Standalone 클러스터 구조와 Master-Worker 방식으로 작업이 분산되는 과정을 이해할 수 있었다.

## 2. Spark Job 실행

구축한 클러스터에서 Spark 애플리케이션을 실행하였다.

- `spark-submit`을 이용해 Spark Job을 제출했다.
- Master가 작업을 Worker에게 분배하고 실행하는 과정을 확인했다.

Spark Job이 클러스터 환경에서 분산 실행되는 방식을 직접 확인할 수 있었다.

## 3. 실행 결과 및 클러스터 확인

Spark Web UI를 통해 클러스터와 작업 상태를 확인하였다.

- Master UI에서 Worker 등록 상태와 가용 자원을 확인했다.
- 실행된 Job과 Stage, Task 정보를 확인하며 작업 진행 과정을 살펴보았다.

이번 미션을 통해 Docker 환경에서 Apache Spark Standalone Cluster를 구축하고, 분산 환경에서 Spark Job이 실행되는 전체 과정을 익힐 수 있었다.
