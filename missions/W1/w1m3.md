# Wikipedia GDP data ETL Pipeline

이번 미션에서는 위키피디아의 국가별 명목 GDP 데이터를 수집하여 ETL(Extract, Transform, Load) 과정을 직접 구현해 보았다. 또한 가공된 데이터를 JSON 파일과 SQLite 데이터베이스에 저장하고, SQL을 이용해 간단한 분석도 수행했다.

## 프로젝트 구성

프로젝트는 다음과 같은 파일로 구성하였다.

- `etl_project_gdp.py` : 위키피디아 데이터를 수집하여 JSON 파일로 저장하는 ETL 스크립트
- `etl_project_gdp_with_sql.py` : JSON 데이터를 SQLite에 저장하고 SQL 분석을 수행하는 스크립트
- `Countries_by_GDP.json` : ETL 수행 후 생성되는 JSON 데이터
- `World_Economies.db` : SQLite 데이터베이스
- `etl_project_log.txt` : ETL 수행 과정을 기록하는 로그 파일

## 이번 미션을 통해 배운 점

### 1. ETL 프로세스의 전체 흐름 이해

ETL 과정을 `Extract`, `Transform`, `Load` 단계로 나누어 구현하면서 데이터 파이프라인의 기본 구조를 이해할 수 있었다.

- **Extract** : 위키피디아에서 GDP 데이터 수집
- **Transform** : 데이터 정제 및 단위 변환
- **Load** : JSON 및 SQLite에 데이터 저장

각 단계를 함수로 분리하여 구현하니 코드의 역할이 명확해지고 유지보수가 쉬워지는 것을 경험했다.

### 2. BeautifulSoup을 활용한 웹 크롤링

BeautifulSoup을 이용하여 HTML 문서에서 원하는 GDP 테이블을 추출했다.

- 여러 개의 `wikitable` 중 필요한 테이블을 찾아 선택했다.
- 테이블의 행과 열을 순회하며 국가명과 GDP 데이터를 추출했다.
- HTML 구조를 먼저 파악한 후 원하는 데이터를 가져오는 과정의 중요성을 배웠다.

### 3. 데이터 전처리

수집한 데이터에는 각주와 쉼표, 괄호 등의 불필요한 정보가 포함되어 있어 이를 정제하는 과정을 수행했다.

- 숫자에 포함된 쉼표와 각주 제거
- GDP 값을 Billion USD 단위로 변환
- 컬럼명을 보다 명확한 형태로 변경
- 국가명에 포함된 각주(`[1]`, `[n 1]` 등)를 제거하여 데이터 품질을 높였다.

데이터를 그대로 사용하는 것이 아니라, 분석하기 좋은 형태로 가공하는 과정이 ETL에서 매우 중요하다는 것을 배웠다.

### 4. SQLite를 활용한 데이터 저장 및 조회

가공된 데이터를 SQLite 데이터베이스에 저장하고 SQL을 이용해 조회를 수행했다.

- `to_sql()`을 사용하여 DataFrame을 SQLite 테이블로 저장했다.
- GDP가 100 Billion USD 이상인 국가를 조회했다.
- 지역(Region)별 상위 5개 국가의 평균 GDP를 SQL로 계산했다.

이를 통해 Pandas에서 처리한 데이터를 데이터베이스에 저장하고, SQL을 이용해 필요한 정보를 조회하는 전체 과정을 경험할 수 있었다.

### 5. 외부 라이브러리를 활용한 데이터 보강

`country_converter` 라이브러리를 활용하여 국가명을 대륙(Region) 정보로 변환했다.

- 대부분의 국가는 자동으로 변환할 수 있었다.
- `Channel Islands`, `Zanzibar`처럼 변환되지 않는 국가는 직접 Region을 지정하여 처리했다.

이를 통해 외부 라이브러리를 사용할 때도 예외 상황을 고려하여 데이터를 보완하는 과정이 필요하다는 것을 배웠다.
