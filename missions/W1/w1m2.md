# SQL 기초 튜토리얼 실습

이번 미션에서는 SQL Tutorial을 따라가며 Jupyter Notebook 환경에서 `sqlite3`를 사용해 기본 SQL 문법을 직접 실행해 보았다. SQL 문법을 익히는 것뿐만 아니라, SQLite를 사용하면서 발생하는 차이점과 한계도 함께 경험할 수 있었다.

## 1. SQLite를 이용한 SQL 실습

이번 과제에서는 별도의 DB 서버를 사용하지 않고, Python의 `sqlite3` 라이브러리를 통해 SQLite 데이터베이스를 생성하고 쿼리를 실행했다.

- `sqlite3.connect()`로 데이터베이스에 연결했다.
- `cursor.execute()`를 사용해 SQL 문을 실행했다.
- `fetchall()`과 `fetchone()`으로 조회 결과를 가져왔다.
- `commit()`을 통해 변경 사항을 저장했다.

이를 통해 Jupyter Notebook 안에서도 간단하게 데이터베이스를 만들고 SQL 실습을 진행할 수 있다는 것을 배웠다.

## 2. SQLite의 특징과 한계

SQLite는 가볍고 사용하기 편리하지만, 다른 RDBMS와 완전히 같은 SQL 문법을 지원하지는 않았다.

- `RIGHT JOIN`, `FULL OUTER JOIN` 같은 조인을 지원하지 않았다.
- `SELECT INTO` 문법도 SQLite에서는 사용할 수 없었다.
- 일부 저장 프로시저 문법도 SQLite에서는 지원되지 않았다.

이 과정에서 SQL 문법은 공통적인 부분이 많지만, 사용하는 DBMS에 따라 지원 여부가 달라질 수 있다는 점을 체감했다.

## 3. SQLite 문법에 맞게 우회하기

지원되지 않는 문법은 SQLite에서 사용할 수 있는 방식으로 바꿔서 해결했다.

예를 들어 `SELECT INTO` 대신 아래와 같은 형태를 사용했다.

```sql
CREATE TABLE US_Customers_Backup AS
SELECT *
FROM Customers
WHERE Country = 'USA';
```

이를 통해 같은 목적의 작업이라도 DBMS에 따라 다른 문법을 사용해야 한다는 것을 배웠다.

## 4. SQL 기본 문법 정리

이번 실습을 통해 기본적인 SQL 문법들을 직접 사용해 보았다.

- SELECT / FROM : 어떤 테이블에서 어떤 컬럼을 조회할지 지정
- WHERE : 조건에 맞는 데이터만 조회
- GROUP BY : 특정 기준으로 데이터 그룹화
- HAVING : 그룹화된 결과에 조건 적용
- ORDER BY : 조회 결과 정렬
- JOIN : 여러 테이블의 데이터 결합
- UNION : 여러 조회 결과를 하나로 합치기
- INSERT / UPDATE / DELETE : 데이터 추가, 수정, 삭제

이번 미션을 통해 SQLite 환경에서 SQL의 기본 문법을 실습했고, 실제로 쿼리를 실행하면서 DBMS마다 문법 차이가 존재한다는 점을 배울 수 있었다.
