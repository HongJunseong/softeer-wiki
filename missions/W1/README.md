# 회고

## SQLite를 사용하며 배운 점

실습을 진행하면서 SQL 문법이 데이터베이스마다 동일하지 않다는 점을 확인할 수 있었다. 다른 DBMS에서 사용할 수 있는 `CREATE PROCEDURE`, `SELECT INTO` 등의 문법은 SQLite에서 지원되지 않았으며, SQLite에 맞는 방식으로 구현해야 했다. 또한 `database is locked` 오류를 경험하며 SQLite가 파일 기반 데이터베이스이기 때문에 동시 접근에 제약이 있다는 점도 이해할 수 있었다.

## ETL 프로세스

BeautifulSoup를 이용해 위키피디아의 GDP 데이터를 수집(Extract)하고, 각주와 불필요한 문자열을 제거한 뒤 단위를 million에서 billion으로 변환하는 전처리(Transform)를 수행했다. 이후 JSON 파일과 SQLite 데이터베이스에 저장(Load)하고, 각 단계의 시작과 종료를 로그 파일에 기록하여 ETL 프로세스의 흐름을 구현했다.

## 데이터 분석

SQLite에 저장된 데이터를 SQL로 조회하며 조건 검색과 데이터 확인을 수행했다. 이 과정에서 컬럼명과 데이터 타입을 정확히 확인하는 것이 중요하다는 점을 배웠고, SQL과 Pandas를 함께 활용하여 데이터를 다루는 방법을 익힐 수 있었다.

## 느낀 점

이번 실습을 통해 데이터를 수집하고, 가공한 뒤, 파일과 데이터베이스에 저장하는 ETL의 전체 흐름을 경험할 수 있었다. 또한 같은 SQL이라도 DBMS마다 지원하는 기능과 문법이 다르다는 점을 직접 확인하며, 사용하는 데이터베이스의 특성을 이해하는 것이 중요하다는 것을 배울 수 있었다.
