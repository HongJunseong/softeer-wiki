# 1주차 회고

## SQLite를 사용하며 배운 점

실습을 진행하면서 SQL 문법이 모든 데이터베이스에서 동일하게 동작하는 것이 아니라는 점을 직접 확인할 수 있었다. 다른 DBMS에서 지원하는 `CREATE PROCEDURE`, `SELECT INTO` 등의 문법은 SQLite에서 사용할 수 없어 SQLite에 맞는 방식으로 구현해야 했다. 또한 `database is locked` 오류를 경험하며 SQLite가 파일 기반 데이터베이스이기 때문에 동시 접근에 제약이 있다는 점도 새롭게 알게 되었다.

## ETL 프로세스

BeautifulSoup를 이용해 데이터를 수집(Extract)하고, 각주 제거와 단위 변환 등 필요한 전처리(Transform)를 거쳐 JSON 파일과 SQLite 데이터베이스에 저장(Load)하는 ETL 과정을 직접 구현했다. 특히 각 단계의 시작과 종료를 로그로 기록하면서 ETL이 단순히 데이터를 옮기는 작업이 아니라, 실행 과정을 추적하고 관리하는 것도 중요한 요소라는 점을 배울 수 있었다.

## 데이터 분석 및 SQL 활용

SQLite에 저장된 데이터를 SQL로 조회하며 조건 검색과 집계 기능을 실습했다. 쿼리가 예상과 다른 결과를 반환했을 때 컬럼명과 데이터 타입을 먼저 확인하는 습관의 중요성을 느꼈고, SQL과 Pandas를 함께 활용하면 데이터를 효율적으로 확인하고 분석할 수 있다는 점도 배울 수 있었다.

## 느낀 점

이번 실습에서는 수업에서 배운 SQL과 ETL 개념을 직접 적용해 보며 이론과 실제 구현의 차이를 경험할 수 있었다. 또한 구현 과정에서 발생한 오류를 해결하기 위해 SQLite 공식 문서와 다양한 자료를 찾아보며 부족했던 부분을 보완할 수 있었다. 앞으로도 수업에서 배운 내용뿐만 아니라, 구현 과정에서 새롭게 알게 된 내용과 문제 해결 과정을 꾸준히 기록하고 관리하는 습관을 가져야겠다고 느꼈다.
