## 회고
### Keep
- pandas를 활용해 데이터를 탐색하고 시각화하며 데이터의 특성을 분석하는 과정을 익혔다.
- SQLite를 이용해 SQL을 직접 실습하고, ETL 과정을 구현하며 데이터 처리 흐름을 이해했다.
- Git과 GitHub를 활용해 브랜치 생성, Push, Pull Request, Merge 과정을 경험하며 협업 방식을 익혔다.
- 오류가 발생했을 때 오류 메시지를 확인하고 원인을 분석하며 해결하는 과정을 반복했다.

### Problem
- SQLite와 다른 DBMS의 문법 차이를 고려하지 않아 일부 SQL 문법이 실행되지 않는 문제가 있었다.
- GitHub 협업 과정에서 Pull Request와 Merge 이후의 흐름을 이해하는 데 어려움이 있었다.
- 새로운 기술을 사용할 때 공식 문서보다 검색 결과에 의존하는 경우가 있어 정확한 정보를 찾는 데 시간이 소요되었다.

### Try
- 새로운 기술이나 라이브러리를 사용할 때는 공식 문서를 먼저 확인하는 습관을 기르겠다.
- SQL을 학습할 때 DBMS별 문법 차이와 지원 범위를 함께 정리하며 학습하겠다.
- Git과 GitHub 협업 과정을 반복 실습하여 브랜치 전략과 Pull Request 흐름을 자연스럽게 익히겠다.
- 단순히 기능을 구현하는 것에 그치지 않고, 각 기술이 왜 필요한지와 실제 활용 사례까지 함께 학습하겠다.

### 느낀 점
1주차 미션을 수행하면서 Git과 GitHub, Jupyter Notebook, Pandas, SQLite 등 데이터를 다루는데 필요한 도구들을 직접 실습하며 익힐 수 있었습니다. 실습 과정에서 발생한 오류도 원인을 찾아 해결해 보며 단순히 따라 하는 것이 아니라 기능과 사용 방법을 이해하려고 노력했습니다. 또한 과제를 수행하면서 필요한 내용을 스스로 찾아보고 적용하며 문제 해결 능력을 키울 수 있었습니다.  

처음 사용하는 개발 환경과 도구가 많아 익숙해지는 데 예상보다 시간이 많이 걸렸습니다. 특히 Git 사용과 SQLite 실습 과정에서 오류가 발생했을 때 해결하는 데 시간이 소요되었습니다. 다음 주차부터는 공식 문서와 자료를 적극적으로 참고하고, 자주 사용하는 명령어와 개념을 미리 정리하여 더 효율적으로 학습하고 과제를 수행하도록 하겠습니다.
