# Hadoop Single-Node Cluster on Docker

## 1. Hadoop 단일 노드 클러스터 구성

이번 미션에서는 Docker를 활용하여 Hadoop 단일 노드 클러스터를 구축하였다.

- Docker Compose를 이용해 Hadoop 컨테이너를 실행했다.
- NameNode와 DataNode가 하나의 환경에서 동작하도록 구성했다.

이를 통해 Hadoop을 로컬 환경에서도 손쉽게 실행하고 테스트할 수 있다는 것을 배웠다.

## 2. Hadoop 환경 설정

Hadoop이 정상적으로 동작하도록 필요한 설정 파일을 구성하였다.

- `core-site.xml`, `hdfs-site.xml`, `mapred-site.xml`, `yarn-site.xml`을 설정했다.
- HDFS와 YARN이 사용할 기본 환경을 이해했다.

설정 파일을 통해 Hadoop의 동작 방식과 클러스터 환경을 정의할 수 있다는 것을 배웠다.

## 3. HDFS 사용

HDFS에 데이터를 저장하고 관리하는 방법을 실습하였다.

- `hdfs dfs` 명령어를 이용해 디렉터리를 생성했다.
- 파일을 업로드하고 목록을 조회하는 방법을 익혔다.
- HDFS와 로컬 파일 시스템이 서로 다른 공간이라는 점을 확인했다.

이를 통해 Hadoop에서는 데이터를 HDFS에 저장하여 관리한다는 점을 이해할 수 있었다.

## 4. Hadoop 서비스 확인

웹 UI를 통해 Hadoop 서비스의 상태를 확인하였다.

- NameNode 웹 UI에서 HDFS 정보를 확인했다.
- YARN 웹 UI에서 클러스터 상태를 확인했다.

명령어뿐만 아니라 웹 UI를 통해 클러스터의 상태와 저장된 데이터를 확인할 수 있다는 것을 배웠다.

## 5. Docker 기반 Hadoop 실행

Docker를 이용해 Hadoop 환경을 관리하는 방법을 익혔다.

- 컨테이너 실행과 종료를 Docker Compose로 관리했다.
- 필요한 설정 파일을 볼륨으로 연결하여 쉽게 수정할 수 있도록 구성했다.

이번 미션을 통해 Docker 환경에서 Hadoop 단일 노드 클러스터를 구축하고, HDFS를 활용하는 기본적인 사용 방법을 익힐 수 있었다.
