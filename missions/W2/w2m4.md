# Multiprocessing Queue All-in-One

## 1. 작업 Queue 구성

이번 미션에서는 여러 프로세스가 하나의 Queue를 공유하여 작업을 수행하는 방법을 학습했다.

- 작업 Queue에 `Task no 0`부터 `Task no 9`까지 저장하였다.
- 완료된 작업을 저장하기 위한 Queue를 별도로 생성하였다.

작업 Queue와 결과 Queue를 분리하여 관리하는 방법을 익힐 수 있었다.

## 2. 여러 프로세스를 이용한 작업 분배

여러 개의 프로세스를 생성하여 작업을 동시에 수행하였다.

- 4개의 프로세스를 생성하였다.
- 각 프로세스가 Queue에서 작업을 하나씩 가져와 수행하도록 구현하였다.

여러 프로세스가 하나의 Queue를 공유하여 작업을 분배하는 방식을 이해할 수 있었다.

## 3. get_nowait()와 예외 처리

Queue가 비었을 때의 처리 방법을 학습하였다.

- `get_nowait()`를 사용하여 즉시 작업을 가져왔다.
- Queue가 비어 있으면 `Empty` 예외를 처리하여 반복을 종료하였다.

예외 처리를 이용하면 남은 작업이 없을 때 안전하게 프로세스를 종료할 수 있다는 것을 배웠다.

## 4. 작업 완료 및 동기화

모든 작업이 완료될 때까지 프로세스를 동기화하였다.

- `join()`을 사용하여 모든 프로세스가 종료될 때까지 기다렸다.
- 완료 Queue에 저장된 결과를 출력하였다.

이번 미션을 통해 여러 프로세스가 Queue를 이용해 작업을 분배하고 처리하는 전체 흐름을 이해할 수 있었다.
