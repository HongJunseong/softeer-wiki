# Docker Image EC2 배포

## 1. Docker 이미지 생성

이번 미션에서는 로컬 환경에서 Docker 이미지를 생성하는 방법을 학습했다.

- `Dockerfile`을 작성하여 애플리케이션 실행 환경을 정의하였다.
- `docker compose build`를 사용하여 이미지를 빌드하였다.
- 필요한 라이브러리와 실행 명령어를 이미지 안에 포함하였다.

Docker 이미지를 사용하면 실행 환경을 하나로 묶어 다른 서버에서도 동일하게 실행할 수 있다는 점을 이해할 수 있었다.

## 2. ECR에 Docker 이미지 업로드

생성한 Docker 이미지를 AWS ECR에 업로드하였다.

- AWS CLI를 이용하여 ECR에 로그인하였다.
- Docker 이미지에 ECR 주소를 태그로 지정하였다.
- `docker compose push` 또는 `docker push`를 사용하여 이미지를 ECR에 업로드하였다.

ECR을 사용하면 Docker 이미지를 AWS 환경에서 안전하게 저장하고 관리할 수 있다는 점을 배웠다.

## 3. EC2에서 이미지 Pull

EC2 서버에 접속하여 ECR에 저장된 Docker 이미지를 가져왔다.

- EC2에 Docker와 Docker Compose를 설치하였다.
- EC2에서도 ECR에 로그인하였다.
- `docker compose pull` 또는 `docker pull`을 사용하여 이미지를 내려받았다.

로컬에서 만든 이미지를 EC2에서 다시 빌드하지 않고 그대로 가져와 실행할 수 있다는 점을 이해할 수 있었다.

## 4. EC2에서 컨테이너 실행

가져온 Docker 이미지를 기반으로 EC2에서 컨테이너를 실행하였다.

- `docker compose up -d`를 사용하여 컨테이너를 백그라운드에서 실행하였다.
- `.env` 파일을 활용하여 환경변수를 주입하였다.
- 컨테이너 실행 상태를 확인하고, 필요한 경우 로그를 확인하였다.

Docker Compose를 사용하면 여러 실행 옵션과 환경변수를 한 파일에서 관리할 수 있어 배포 과정이 더 편리하다는 점을 배웠다.

## 5. 배포 과정 정리

이번 배포 과정은 로컬에서 이미지를 만들고, ECR을 거쳐 EC2에서 실행하는 흐름으로 진행되었다.

- 로컬에서 Docker 이미지 빌드
- ECR에 이미지 Push
- EC2에서 이미지 Pull
- EC2에서 Docker Compose로 컨테이너 실행

이번 미션을 통해 Docker 이미지를 직접 EC2에 배포하면서, 로컬 개발 환경과 서버 실행 환경을 분리하고 동일한 애플리케이션을 안정적으로 실행하는 전체 흐름을 경험할 수 있었다.
